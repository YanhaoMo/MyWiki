# 目录结构
假设我们现在有一个名字叫做 foo 的项目，那么 foo 一定是这个项目的顶层目录。
在 foo 目录下，一般需要包含这些文件或目录：

- README 
- src
- include
- build
- test
- Makefile

以上是必须包含的，顺便，下面是一些可选项目：

- LICENSE
- doc

# 头文件
一般来讲，项目中(src目录下)的每一个源文件，
都需要在include目录中有一个头文件与其对应。
头文件是其他文件使用该源文件中函数的接口。同时，
在该源文件自身中也应该包含自己的头文件，
这样可以防止源代码中出现函数声明与实现不一致的错误，
而这种错误是不容易被编译器发现的。
